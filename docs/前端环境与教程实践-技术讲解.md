## 1. npm 与前端工程结构

- **npm**（Node Package Manager）随 Node.js 安装，负责：
  - **安装依赖**：根据 **`frontend/package.json`** 把依赖下载到 **`frontend/node_modules`**。
  - **执行脚本**：运行 **`package.json` → `scripts`** 里的条目（如开发服务器、构建、OpenAPI 生成）。
- **类比（便于从 Python 过渡）**：
  - `package.json` ≈ `pyproject.toml` / 依赖清单  
  - `npm install` ≈ `pip install -e .` / `uv sync`  
  - `npm run dev` ≈ 启动本地开发服务（本项目用 **Vite**）

## 2. 本项目前端技术栈（与编程导航教程对齐）

| 层次 | 选型 | 作用 |
|------|------|------|
| 框架 | Vue 3 | 组件化 UI |
| 构建 / 开发服务器 | Vite | 快速冷启动、热更新 |
| 语言 | TypeScript | 类型约束 |
| 路由 | Vue Router | 页面路径与懒加载 |
| 状态 | Pinia | 全局状态（后续登录用户等） |
| UI | Ant Design Vue 4 | 表单、布局、反馈组件 |
| HTTP | Axios | 请求；项目在 **`src/request.ts`** 统一实例化 |
| 接口类型（可选） | `@umijs/openapi` | 从 OpenAPI 文档生成 TS 与请求函数 |

### 请求与登录（Session）

- 浏览器只访问 **前端开发服务器**（如 `http://localhost:5174`），页面里的接口路径写成 **`/api/...`**。
- **Vite** 在开发模式下把 **`/api`** **代理**到本机后端 **`8567`**，避免手写完整域名、减轻跨域配置负担。
- 后端使用 **Cookie（SESSION）** 存会话时，Axios 必须 **`withCredentials: true`**，否则不会带上 Cookie，登录态无效。
- 后端统一返回 **`{ code, data, message }`**；业务上 **`code === 40100`** 表示未登录（与 **`request.ts`** 里跳转登录页的逻辑一致）。

### OpenAPI 与 Spring / FastAPI 的差异

- 教程原文若来自 Spring，文档地址常为 **`/v3/api-docs`**。
- 本项目后端为 **FastAPI**，默认 OpenAPI JSON 一般为：**`http://127.0.0.1:8567/openapi.json`**（生成脚本配置见 **`frontend/openapi2ts.config.ts`**）。

## 3. 前后端如何协同

| 角色 | 典型地址 | 说明 |
|------|-----------|------|
| 前端开发服务 | `http://localhost:5173`，占线时可能是 **5174、5175…** | 终端里 **Local** 为准 |
| 后端 API | `http://127.0.0.1:8567` | 单独终端常驻 |
| 用户浏览器 | 打开 **前端** 的 URL | 数据请求走 **`/api`**，由 Vite 转到后端 |

**CORS**：浏览器会校验「前端页面来源」与「接口响应是否允许该来源」。本项目在 **`python-backend/app/main.py`** 中对本地开发使用 **`localhost` / `127.0.0.1` 任意端口** 放行，并允许 **`credentials`**，这样在 Vite 换端口时 Cookie 仍可用。

## 5. Windows 环境问题（原理简述）

| 现象 | 常见原因 |
|------|-----------|
| Cursor / PowerShell 里找不到 `node`、`npm` | 集成终端启动时未继承最新 **PATH**（安装 Node 后未重启 IDE） |
| CMD 可用、PowerShell 不可用 | 同上；或当前会话 Path 未包含 **`Program Files\nodejs`** |
| **nvm-windows** 报目录不存在、`nvm use` 失败 | **NVM_ROOT** 路径无效、中文路径、或 **`NVM_SYMLINK`**（常见 **`C:\Program Files\nodejs`**）未创建 —— 往往需在 **管理员 CMD** 下 **`nvm use`** |
| 官方安装 Node 与 nvm 并存 | **`where node`** 可能出现多条路径，注意当前生效顺序 |

**端口**：若 **5173** 已被占用（例如上一次 **`npm run dev`** 未关），Vite 会自动递增端口；只要后端 CORS 覆盖本地任意端口，换端口不影响联调。
