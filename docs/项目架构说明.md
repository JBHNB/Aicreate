# AI 爆款文章创作器 — 项目架构说明（Python 版）

> 仓库：[JBHNB/Aicreate](https://github.com/JBHNB/Aicreate)  
> 对标官方教程：[yuyuanweb/ai-passage-creator](https://github.com/yuyuanweb/ai-passage-creator)（Java / Go / Python）  
> **本仓库 = FastAPI + Vue 3 + DashScope**，业务与官方一致，技术栈为 Python 实现。

---

## 一、整体架构（五层设计）

系统采用**分层架构 + 多智能体编排**，用户输入选题后，由多个 Agent 协作完成标题 → 大纲 → 正文 → 配图 → 图文合成，各阶段支持人机协作。

| 层级 | 技术 | 职责 |
|------|------|------|
| **接入层** | Vue 3 + Vite + Nginx | 前端渲染、静态资源、反向代理 `/api` |
| **接口层** | FastAPI | RESTful API、SSE 流式推送、Depends 鉴权 |
| **业务层** | Service 模块 | 用户 / 文章 / 支付 / 统计 / 配图策略 |
| **智能体层** | Orchestrator + Agents | 5+1 Agent 协作、并行配图、流式输出 |
| **数据层** | MySQL + Redis + COS | 业务数据、Session 缓存、图片对象存储 |

## 二、各层说明

### 1. 接入层

| 组件 | 路径 | 说明 |
|------|------|------|
| 前端 | `frontend/` | Vue 3 + TypeScript + Pinia + Ant Design Vue |
| 创作页 | `ArticleCreatePage.vue` | 三栏布局、SSE 进度、阶段确认 |
| 首页 | `HomePage.vue` | 营销风 Hero + 最近创作 |
| 管理统计 | `admin/StatisticsPage.vue` | ECharts 运营看板 |
| 部署 | `frontend/Dockerfile` + `nginx.conf` | 生产环境 Nginx 反代 `/api` |

开发时 Vite `:5173` 通过 proxy 转发到 FastAPI `:8569`（或 `:8567`）。

### 2. 接口层（FastAPI Routers）

| 路由前缀 | 文件 | 主要接口 |
|----------|------|----------|
| `/api/user` | `routers/user.py` | 注册、登录、登出、当前用户 |
| `/api/article` | `routers/article.py` | 创建、SSE 进度、确认标题/大纲、列表 |
| `/api/passage` | `routers/passage.py` | 简易单次创作（无多阶段） |
| `/api/payment` | `routers/payment.py` | Stripe VIP 会话 |
| `/api/webhook/stripe` | `routers/payment.py` | 支付回调 |
| `/api/statistics` | `routers/statistics.py` | 管理员统计概览 |
| `/api/health` | `routers/health.py` | 健康检查 |

**鉴权：** `deps.py` 中 `require_login` / `require_admin`；Cookie 只存 Session UUID，用户在 Redis `session:{uuid}`，支付后从 MySQL 刷新 role。

**SSE：** `GET /api/article/progress/{taskId}` → `sse_emitter_manager` 长连接推送 Agent 进度。

### 3. 业务层（Services）

| 服务 | 文件 | 职责 |
|------|------|------|
| **ArticleService** | `article_service.py` | 文章 CRUD、配额、phase 状态机、confirm 标题/大纲 |
| **ArticleAsyncService** | `article_async_service.py` | phase1/2/3 长任务、`asyncio.create_task`、SSE 消息桥接 |
| **ArticleAgentService** | `article_agent_service.py` | OpenAI/DashScope 客户端、各 Agent 实际 LLM 调用、占位符归一化 |
| **ImageServiceStrategy** | `image_service_strategy.py` | 策略模式注册 Pexels / Mermaid / Iconify / 表情包 / Nano / SVG |
| **PaymentService** | `payment_service.py` | Stripe Checkout Session、Webhook 升 VIP |
| **StatisticsService** | `statistics_service.py` | 今日/本周/本月创作、成功率、Redis 缓存 |
| **AgentLogService** | `agent_log_service.py` | 智能体执行日志、耗时统计 |
| **UserService** | `user_service.py` | 用户注册登录、密码加盐 MD5 |
| **PassageService** | `passage_service.py` | 简易 Passage 创作（教程第 2 期） |

**三个 Article 服务的分工：**

```text
Router
  → ArticleService        （同步：CRUD、配额、confirm）
  → ArticleAsyncService   （异步导演：phase 编排 + SSE）
      → ArticleAgentService （AI 工人：LLM + 配图 + state）
          → ArticleAgentOrchestrator + Agents
```

### 4. 智能体层

| Agent | 类 | 阶段 | 说明 |
|-------|-----|------|------|
| Agent 1 | `TitleGeneratorAgent` | phase1 | 生成 3–5 个标题方案 |
| Agent 2 | `OutlineGeneratorAgent` | phase2 | 流式生成大纲 |
| Agent 3 | `ContentGeneratorAgent` | phase3 | 流式生成 Markdown 正文 |
| Reviewer | `ReviewerAgent` | phase3 | 正文质量审核（可配置关闭） |
| Agent 4 | `ImageAnalyzerAgent` | phase3 | 分析配图需求、插入占位符 |
| Agent 5 | `ParallelImageGenerator` | phase3 | **asyncio.Semaphore** 并行配图 |
| 合成 | `ContentMergerAgent` | phase3 | `{{IMAGE_PLACEHOLDER_N}}` 替换为图片 |

**编排器：** `app/agent/orchestrator.py` — `ArticleAgentOrchestrator`  
**流式上下文：** `app/agent/context/stream_handler.py` — `StreamHandlerContext.emit()`  
**并行配图：** `app/agent/parallel/image_generator.py` — `asyncio.gather` + 信号量控并发  

> 与 Java 版区别：Java 用 Spring AI **StateGraph**；Python 版用 **Orchestrator + Agent 适配器**，语义等价。

### 5. 数据层

| 存储 | 用途 | 关键表/Key |
|------|------|------------|
| **MySQL** | 业务持久化 | `user`、`article`、`agent_log`、`payment_record`、`passage` |
| **Redis** | Session | `session:{uuid}` → 用户快照 |
| **Redis** | 统计缓存 | `statistics:overview:v2` |
| **腾讯云 COS** | 配图 URL | 各 ImageSearchService 上传后返回 CDN 地址 |

**文章表关键字段：** `taskId`、`phase`、`status`、`titleOptions`、`outline`、`content`、`fullContent`、`images`、`enabledImageMethods`

## 三、交互流程

### 3.1 一次完整创作

```text
用户登录
  → POST /api/article/create { topic, style, imageMethods }
  → ArticleService 扣配额 + INSERT article（PENDING）
  → 立即返回 taskId

并行：
  前端 GET /api/article/progress/{taskId}  （SSE 长连接）
  后端 asyncio.create_task(execute_phase1)

phase1：Orchestrator → TitleAgent → 保存 titleOptions → SSE: TITLES_GENERATED
  → 用户 POST confirm-title
phase2：Orchestrator → OutlineAgent 流式 → save outline → SSE: OUTLINE_GENERATED
  → 用户 POST confirm-outline
phase3：Content → Reviewer? → ImageAnalyzer → ParallelImage → Merger
  → save fullContent → status=COMPLETED → SSE: ALL_COMPLETE
```

### 3.2 阶段状态机（article.phase）

```text
PENDING → TITLE_GENERATING → TITLE_SELECTING
  → OUTLINE_GENERATING → OUTLINE_EDITING
  → CONTENT_GENERATING →（status=COMPLETED）
```

### 3.3 SSE 消息类型

| 类型 | 含义 |
|------|------|
| `AGENT1_COMPLETE` | 标题方案完成 |
| `AGENT2_STREAMING` | 大纲流式片段 |
| `AGENT2_COMPLETE` | 大纲完成 |
| `AGENT3_STREAMING` | 正文流式片段 |
| `AGENT_REVIEWER_COMPLETE` | 审核完成 |
| `AGENT4_COMPLETE` | 配图需求完成 |
| `IMAGE_COMPLETE` | 单张配图完成 |
| `MERGE_COMPLETE` | 图文合成完成 |
| `ALL_COMPLETE` | 全流程结束 |
| `ERROR` | 失败 |

## 四、配图策略模式

```text
ImageServiceStrategy
  ├── PexelsService        （关键词搜图）
  ├── MermaidService       （流程图 SVG）
  ├── IconifyService       （图标）
  ├── EmojiPackService     （表情包 Bing）
  ├── NanoBananaService    （AI 生图 · VIP）
  ├── SvgDiagramService    （AI 示意图 · VIP）
  └── PicsumService        （降级随机图）
```

主方式失败时自动降级 Picsum，保证创作不中断。

## 七、面试 30 秒版

> 我做的是 AI 爆款文章创作器，Python FastAPI + Vue 3。核心是**五层架构 + 三阶段人机协作**：用户创建任务后立即拿 taskId，后台用 asyncio 跑多 Agent 流水线，进度通过 SSE 推前端。智能体层有 Orchestrator 编排 5 个 Agent，配图用策略模式支持 6 种来源，并行生成用 Semaphore 控并发。业务上 ArticleService 管 DB 和状态机，ArticleAsyncService 管长任务和 SSE，ArticleAgentService 管 LLM。Session 在 Redis，支付接 Stripe Webhook 升 VIP。
