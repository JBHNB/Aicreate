# 前后端联调与实现技术总结

本文档汇总「前后端如何协作、各层如何实现、常见环境与技术概念」，便于复习与 onboarding。与《项目功能与开发流程（新手向）》互补：前者偏**跑项目与目录地图**，本文偏**原理与联调细节**。

---

## 一、技术栈与职责

| 层级 | 技术 | 职责 |
|------|------|------|
| 浏览器 UI | Vue 3 + Vite + Ant Design Vue | 页面、路由、状态、调用 HTTP |
| 开发代理 | Vite `server.proxy` | 开发时把 `/api` 转发到后端，减少跨域问题 |
| API 服务 | FastAPI（Uvicorn） | 路由、鉴权、业务编排、统一响应与异常 |
| 持久化 | MySQL | 用户、文章等业务数据 |
| 会话存储 | Redis | Session 数据（登录态），键形如 `session:{uuid}` |
| 配置 | `.env` + `pydantic-settings` | 数据库、Redis、Session 等环境相关配置 |

---

## 二、开发时请求实际怎么走

1. 浏览器打开前端：`http://localhost:5173`（Vite 端口可能为 5174、5175…）。
2. 前端 Axios `baseURL` 为 **`/api`**，请求发往**同源**的 `http://localhost:5173/api/...`。
3. **`vite.config.ts`** 中配置 `proxy['/api'] → http://127.0.0.1:8567`，请求被转发到 FastAPI。
4. FastAPI 在 **`main.py`** 里为路由统一加了 **`prefix="/api"`**，例如用户登录完整路径为 **`POST /api/user/login`**。

前端 `request.ts` 要点：

- **`withCredentials: true`**：跨域或代理场景下尽量携带 **Cookie**（与登录 Session 配套）。
- **响应拦截**：若业务码 **`code === 40100`**（未登录），可跳转登录页（避免用户在受保护页空白报错）。

后端 CORS（`main.py`）要点：

- **`allow_credentials=True`**：允许带 Cookie 的跨域请求（需与具体 `Origin` 配合，不能 `*`）。
- 可使用 **`allow_origin_regex`** 匹配 `localhost` / `127.0.0.1` 多端口，避免 Vite 改端口就要改后端。

## 三、前后端「合同」：统一 JSON 结构

约定响应体形态：

```json
{ "code": 0, "data": ..., "message": "ok" }
```

- **`code === 0`**：业务成功。
- **`code !== 0`**：业务失败（如 40100 未登录）；很多场景下 HTTP 状态码仍为 200，由前端根据 `code` 分支处理。

后端：`app/schemas/common.py` 中 **`BaseResponse`**；业务异常用 **`BusinessException`**，在 `main.py` 中 **`@app.exception_handler`** 转成上述 JSON。

## 四、登录态：Cookie、Session、Redis 的关系

| 概念 | 是什么 | 在本项目中的角色 |
|------|--------|------------------|
| **Cookie** | 浏览器保存的小段键值 | 只存 **`SESSION=<随机 UUID>`**，由登录接口 **`response.set_cookie`** 写入 |
| **Session（机制）** | 用「钥匙」换服务端状态 | **钥匙** = Cookie 里的 UUID；**状态** = Redis 里 JSON |
| **Redis** | 独立内存数据库 | 存 **`session:{uuid}` → `{"user": {...}}`**，带过期时间 |

流程简述：

1. **登录成功**：生成 `session_id` → **`set_session`** 写 Redis → **`set_cookie(SESSION=session_id)`**。
2. **后续请求**：浏览器自动带 **`Cookie: SESSION=...`**（在规则允许且 `withCredentials` 打开时）→ **`deps.get_session_id`** 读 Cookie → **`get_session`** 读 Redis → 得到 **`LoginUserVO`**。
3. **登出**：删 Redis 中 key + **`delete_cookie`**。

**注意**：`deps.py` 只负责**读**会话；写 Cookie / 写 Redis 在 **`app/routers/user.py`** 的登录逻辑中。

## 五、后端分层（实现顺序心智模型）

```
main.py          → 应用入口：生命周期连接 MySQL/Redis、CORS、全局异常、挂载路由
routers/*.py     → HTTP：路径、Depends、调 Service、返回 BaseResponse
deps.py          → 鉴权链：Cookie → Redis → require_login / require_admin
services/*.py   → 业务：校验、SQL、不直接处理 HTTP
database.py      → SQLAlchemy 同步引擎 + databases 异步连接
models/*.py      → ORM 表结构（与 SQL 对齐）
schemas/*.py     → Pydantic：请求/响应形状、校验、alias 驼峰
utils/session.py → Redis 读写 Session
config.py        → Settings：含 database_url、async_database_url、redis_url 等 @property
```

**示例：文章归属当前用户**——路由里 **`Depends(require_login)`** 得到 **`current_user`**，调用 **`PassageService.generate_and_save(current_user.id, request)`**，插入 **`userId`**；**不要**信任前端 body 里传来的「用户 id」作为归属依据。

## 六、Pydantic：`alias` 与 `response_model_by_alias`

- Python 属性常用 **蛇形**：`user_id`、`create_time`。
- 前后端 JSON 常用 **驼峰**：`userId`、`createTime`。
- **`Field(..., alias="userId")`**：对外 JSON 用别名；代码里仍用 `user_id`。
- **`model_config = ConfigDict(populate_by_name=True)`**：解析时兼容别名与字段名。
- 路由上 **`response_model_by_alias=True`**：FastAPI **序列化响应**时使用 alias，与前端字段一致。

---

## 七、FastAPI：`Depends` 与 `deps.py`

- **`Depends(get_current_user)`**：可选登录，未登录返回 `None`。
- **`Depends(require_login)`**：必须登录，否则抛 **`NOT_LOGIN_ERROR`**。
- **`Depends(require_admin)`**：必须先登录且 **`user_role == "admin"`**。

依赖可嵌套，形成链：**`get_session_id` → `get_current_user` → `require_login` → `require_admin`**。

## 九、环境与依赖（易踩坑）

### Python 依赖装在哪

- **`pyproject.toml`** 只声明依赖，**不表示已安装**。
- 本项目常用：**`python-backend/.venv`** 下执行 **`pip install -e .`**。
- 在仓库根目录的 **conda** 里执行 **`pip show fastapi`** 若提示未找到，多半是**查错了环境**；应 **`cd python-backend`** 后激活 **`.venv`** 或使用 **`.venv\Scripts\pip.exe`**。

### `uv`

- 教程中的 **`uv sync`** 需要本机已安装 **uv**；未安装时用 **`pip install -e .`** 即可，不必强求 uv。

### MySQL 与 DBeaver

- **DBeaver 是客户端**，不能代替 MySQL 服务；需本机安装 MySQL、Docker 跑 MySQL，或云数据库。
- **Docker**：可用 **`docker compose`** 启动 `mysql` + `redis`（见 `python-backend/docker-compose.yml`）。
- **DBeaver 连接 MySQL 8**：若报 **`Public Key Retrieval is not allowed`**，在驱动属性中加 **`allowPublicKeyRetrieval=true`**。
- **执行 SQL**：需先 **`CREATE DATABASE`** 并 **`USE`** 或工具栏选中库，否则 **`No database selected`** / **`Unknown database`**。
- 表名 **`user`** 为 MySQL 保留字，SQL 中建议写作 **`` `user` ``**。

### 异步数据库 URL

- **`databases`** 使用 **`mysql://...`**（`aiomysql`），与 SQLAlchemy 的 **`mysql+pymysql://...`** 不同；项目在 **`config.async_database_url`** 中做替换。

## 十、联调自检清单

1. **Docker**：`docker compose ps` 中 MySQL、Redis 为运行中（若使用 compose）。
2. **`.env`**：`DB_*`、`REDIS_*` 与真实服务一致。
3. **后端**：`uvicorn app.main:app --reload --host 0.0.0.0 --port 8567`；访问 `/api/health` 或 `/docs`。
4. **前端**：`npm run dev`；Network 中请求为 **`/api/...`**，登录后后续请求带 Cookie（视代理与 SameSite 策略而定）。
5. **业务**：`code === 0` 表示成功；40100 触发前端登录跳转逻辑。

## 十一、扩展新功能时的推荐顺序

1. 定接口（路径、方法、是否登录、请求/响应 JSON）。
2. 数据库变更（`sql/` 或迁移脚本）。
3. 后端：`schemas` →（可选）`models` → `services` → `routers`，并在 `main.py` 注册路由。
4. Swagger 或 curl 自测；需登录接口先调 login。
5. 前端：页面 + `request` 调用；处理 `code` / 错误提示。
