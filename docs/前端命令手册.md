# 前端命令手册（可复制）

仅列命令与最短说明；原理见 **`前端环境与教程实践-技术讲解.md`**。  
**完整从零搭建流程**见 **[从零开发手册.md](./从零开发手册.md)**。

**约定**：工作区根目录 `D:\agi_code\aicreate`；前端目录 **`frontend`**；后端目录 **`python-backend`**。请在 **`npm` 可用的终端**执行（推荐 CMD，或已刷新 PATH 的 PowerShell）。

---

## 前端（`frontend`）

进入目录：

```bat
cd /d D:\agi_code\aicreate\frontend
```

安装依赖（首次或 `package.json` 变更后）：

```bat
npm install
```

安装单个依赖示例：

```bat
npm i --save 包名
npm i --save-dev 开发依赖包名
```

启动开发服务器（保持窗口不关）：

```bat
npm run dev
```

浏览器打开终端输出的 **Local** 地址（如 `http://localhost:5174`）。

### 多阶段创作（对齐官方教程第 3 节前端）

| 路径 | 说明 |
|------|------|
| `/article/create` | 创作页：选题 → 标题 → 大纲 → 正文（SSE 流式） |
| `/article/list` | 创作历史：搜索、查看、导出、删除 |
| `/article/:taskId` | 文章详情：Markdown 预览、执行日志、导出 |
| `/admin/statistics` | 数据分析（管理员） |

旧路径 `/article/workspace` 会自动跳转到 `/article/create`。

TypeScript 检查：

```bat
npm run typecheck
```

生成接口代码（**须先启动后端 8567**）：

```bat
npm run openapi2ts
```

输出目录：`frontend/src/services`。

生产构建：

```bat
npm run build
```

## 后端（与前端并行，新开终端）

```bat
cd /d D:\agi_code\aicreate\python-backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8567
```

**重要**：前端通过 `frontend/.env.development` 的 `VITE_API_TARGET` 代理到后端，**前后端口必须一致**。

默认 `http://127.0.0.1:8567`。若启动报 **WinError 10048** 且 `taskkill` 提示找不到进程（8567 幽灵占用），请改用 **8569**：

```bat
python -m uvicorn app.main:app --host 0.0.0.0 --port 8569
```

并确认 `frontend/.env.development` 为：

```env
VITE_API_TARGET=http://127.0.0.1:8569
```

改端口后需**重启** `npm run dev`。

多阶段文章创作耗时较长，**不要用 `--reload`**，否则后台任务会被热重载打断。

接口文档：<http://localhost:8567/docs>

## PowerShell：刷新 PATH（Cursor 里找不到 npm 时）

```powershell
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
```

临时使用 npm 全路径示例：

```bat
"C:\Program Files\nodejs\npm.cmd" -v
```

## nvm-windows（按需）

查看版本与根目录：

```bat
nvm list
nvm root
```

切换版本（示例）：

```bat
nvm use 22.12.0
```

目录异常时可（管理员 CMD）：

```bat
mkdir C:\nvm
nvm root C:\nvm
nvm install 22.12.0
nvm use 22.12.0
```
